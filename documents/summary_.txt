The document presents a novel multilevel classifier fusion approach called DroidFusion for improving Android malware detection using machine learning techniques.
Android Malware Detection Overview ​
Android malware is increasingly sophisticated and poses significant security threats to mobile devices. ​ The paper introduces a novel machine learning-based classifier fusion approach called DroidFusion to enhance malware detection accuracy. ​
DroidFusion Framework and Methodology ​
DroidFusion employs a multilevel architecture for classifier fusion, combining various machine learning algorithms to improve detection accuracy. ​

The framework trains base classifiers at a lower level and applies ranking-based algorithms at a higher level to derive a final classifier. ​
It can utilize both traditional classifiers and ensemble learning algorithms for enhanced performance. ​
The approach was evaluated on four datasets, demonstrating its effectiveness in Android malware detection. ​

Experimental Results and Comparisons ​
Extensive experiments were conducted to validate the effectiveness of the DroidFusion approach against other methods.

The results showed that DroidFusion outperformed traditional classifiers and ensemble methods, achieving higher accuracy rates. ​
The framework was compared to stacked generalization, a well-known classifier fusion method, and demonstrated superior performance.

Related Work in Android Malware Detection ​
The paper reviews existing methods in Android malware detection, highlighting the use of static and dynamic analysis techniques. ​

Static analysis is faster but vulnerable to obfuscation, while dynamic analysis is more resistant but can be limited by anti-virtualization techniques. ​
Previous works have explored various machine learning algorithms, including SVM, random forest, and naive Bayes, with varying degrees of success. ​

Classifier Fusion Techniques in Malware Detection ​
The paper discusses the application of classifier fusion techniques in improving malware detection accuracy. ​

Previous studies have utilized methods like majority voting and product of probabilities to combine classifiers. ​
The proposed DroidFusion framework introduces four ranking-based algorithms to enhance the selection and combination of classifiers for better performance.

Automated Static Analyzer for Feature Extraction ​
The DroidFusion system utilizes an automated static analysis tool to extract features from Android applications for malware detection. ​ This tool extracts permissions, intents, API calls, and searches for dangerous commands, providing essential features for machine learning models.

The static analysis tool is developed in Python. ​
It extracts permissions and intents from the Android manifest file using AXMLprinter2. ​
API calls are obtained by reverse engineering .dex files with Baksmali disassembler. ​
The tool checks for dangerous Linux commands and embedded files (.dex, .jar, .so, .exe). ​
Features are represented in binary form and labeled with class values.
Excludes third-party libraries using a list of popular ad libraries. ​

Feature Selection and Ranking Methodology
Feature selection is crucial for reducing dimensionality and improving model efficiency, with the study applying information gain to rank and select features. ​ The initial set of 350 features was reduced to 100 for the experiments.

Four datasets were used for evaluation, including Malgenome-215 and Drebin-215. ​
Information Gain (IG) is used to rank features based on their significance. ​
The formula for IG is IG = E(X) - E(X/Y), where E(X) is the entropy of feature X. ​
The study reduced features from 350 to 100 using IG, enhancing model performance. ​

Model Evaluation Metrics for Performance Assessment
The evaluation of the models is based on several performance metrics, including True Positive Rate (TPR), False Positive Rate (FPR), Precision, F-Measure, and testing time. ​ These metrics provide a comprehensive view of the model's predictive power.

TPR measures the ratio of correctly classified malicious apps. ​
FPR assesses the ratio of incorrectly classified benign instances. ​
Precision indicates the positive predictive rate. ​
F-Measure combines precision and recall, especially useful for unbalanced datasets. ​
Time taken to test the model is also recorded, with specific times noted for different datasets.

Datasets Used for DroidFusion Evaluation ​
DroidFusion was evaluated using four datasets derived from three collections of Android app samples, with detailed characteristics provided for each dataset. ​

Malgenome-215: 3799 app samples (2539 benign, 1260 malware), 215 features. ​
Drebin-215: 15,036 app samples (9476 benign, 5560 malware), also 215 features. ​
McAfee-350: 36,183 instances (13,805 malware, 22,378 benign), 350 features.
McAfee-100: Derived from McAfee-350, reduced to 100 features.

Performance Evaluation of DroidFusion on Malgenome-215 ​
DroidFusion's performance was evaluated on the Malgenome-215 dataset, demonstrating significant improvements over individual base classifiers. ​

The dataset was split into 80% training-validation and 20% testing.
J48 classifier achieved the best malware recall of 0.975 and benign recall of 0.983. ​
The best combination scheme (AAB+RAPC) achieved a weighted F-measure of 0.9893. ​
DroidFusion outperformed traditional combination methods and individual classifiers on the test set.

Performance Evaluation of DroidFusion on Drebin-215 ​
The evaluation of DroidFusion on the Drebin-215 dataset showed superior performance compared to base classifiers and traditional methods.

The dataset was split into 90% training-validation and 10% testing. ​
Random tree-100 had the best malware recall of 0.968, while J48 had the best benign recall of 0.983. ​
The final DroidFusion model achieved a weighted F-measure of 0.9872, outperforming all base classifiers. ​

Performance Evaluation of DroidFusion on McAfee-350 ​
DroidFusion's performance on the McAfee-350 dataset indicated improvements over individual classifiers and traditional combination schemes. ​

The training-validation/testing split was consistent with previous experiments. ​
J48 had the highest benign recall of 0.973, while random tree-100 had the best malware recall of 0.948. ​
The weighted F-measure for J48 was 0.9684, lower than combination schemes. ​
DroidFusion's results were comparable to majority vote and average probabilities but surpassed maximum probability methods. ​

Performance Evaluation of DroidFusion on McAfee-100 with Ensemble Learners ​
The experiments with ensemble classifiers aimed to assess the effectiveness of DroidFusion in enhancing accuracy through ensemble learning. ​

The number of features was reduced from 350 to 100 using IG.
Five ensemble models were tested, including random forest and AdaBoost. ​
The final DroidFusion model achieved the highest weighted F-measure, improving benign recall rates over all ensemble models. ​

Comparison of DroidFusion with Stacked Generalization ​
DroidFusion was compared to stacked generalization, revealing its advantages in classifier fusion without the need for a meta-learner.

Stacked generalization used a linear regression meta classifier for fusion.
DroidFusion outperformed stacked generalization in three out of four datasets. ​
The flexibility of DroidFusion allows for a wider range of criteria for weighting and fusion.

Analysis of Time Performance for Feature Extraction ​
The processing time for feature extraction and analysis was evaluated, demonstrating the efficiency of the DroidFusion system. ​

Average unzipping and disassembly time was 0.739 seconds. ​
Analyzing the manifest and extracting features took approximately 0.0048 seconds.
Total average processing time for apps was around 7.145 seconds. ​
Testing times for DroidFusion varied based on the number of instances, showcasing scalability. ​

Limitations and Future Work for DroidFusion ​
While DroidFusion has shown improved accuracy, there are limitations, particularly in its current design for binary classification. ​

Future work could extend the framework to handle multiclass problems.
The general-purpose nature of DroidFusion allows for potential enhancements in various applications. ​
Further research is needed to explore additional features and classifiers for improved performance.
